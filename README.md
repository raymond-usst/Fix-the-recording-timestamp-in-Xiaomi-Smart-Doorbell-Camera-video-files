# 小米智能门铃批量重命名工具 (Auto Rename UI)

这是一个基于 Python 的带有图形用户界面 (GUI) 的智能化视频批量重命名工具，专为行车记录仪视频设计。它通过截取视频帧并利用本地 AI (EasyOCR) 识别画面上的时间水印，将毫无规律的视频源文件准确重命名为录制时间戳，并同步修改文件的系统创建/修改时间以及底层多媒体元数据 (Metadata)。

## ✨ 核心特性 / Features

1. **AI 视觉识别 (OCR)**
   - 使用 `easyocr` 引擎识别视频画面中的时间戳。
   - 自动截取视频顶部区域（常包含行车记录仪时间水印），并对画面进行灰度化和放大增强处理，以提升 OCR 识别精度。
2. **精准的时间校准**
   - 提取识别到时间戳的帧位置（如第 5 秒），并在计算初始启动录制时间时，自动**减去该时间差**，还原出最真实的视频开始录制时间。
3. **元数据与文件属性全面同步**
   - **重命名格式**：`YYYY-MM-DD_HH-MM-SS.ext`
   - **底层元数据修改**：通过 `ffmpeg` (`imageio_ffmpeg`) 写入底层的 `creation_time`。
   - **Windows 属性修改**：通过 `win32_setctime` 和 `os.utime` 修改系统级别的文件创建时间和修改时间。确保在任何播放器或资源管理器中按时间排序都绝对正确。
4. **流畅优雅的 GUI 体验**
   - 基于 `tkinter` (`ttk`) 打造的现代化、拟物化窗口设计。
   - 提供 Apple 风格的 Ease-Out 平滑进度条动画。
   - 科学精准的 ETA (预计剩余时间) 算法：基于**数据流量(Bytes/s)**而不是文件个数进行计算，即便遇到不同大小的视频，依然能给出高度准确的预估倒计时。
5. **智能分辨率标记后缀** (可选)
   - 开启特定参数时，可根据视频分辨率自动追加后缀，例如：
     - `1920x1080` -> `_verticalview`
     - `2672x1728` -> `_frontview`

## 📦 依赖库 / Prerequisites

运行该工具需要 Python 3.7+ 以及以下第三方库的支撑：

```bash
pip install opencv-python easyocr tk imageio-ffmpeg win32-setctime
```

*注意：`easyocr` 在首次运行时会自动下载 OCR 语言模型（若无）。推荐在有 NVIDIA GPU 的机器上使用并安装对应的 PyTorch 版本以启用 CUDA 加速，显著提升识别和处理速度。*

## 🚀 使用方法 / Usage

### 目录结构要求
默认情况下，代码中固定了输入和输出文件夹路径。请在代码所在根目录下确保以下文件夹结构存在：

- **输入目录**: `D:\code\auto_rename by ocr pic\vid` （请将待处理的行车记录仪视频放入此目录）
- **输出目录**: `D:\code\auto_rename by ocr pic\result` （处理后的视频将会输出到这里）

### 运行程序

直接运行 Python 脚本以启动图形界面：

```bash
python auto_rename_ui.py
```

如果你希望工具能够自动根据视频分辨率（1080p 或特定分辨率）追加视图后缀名称，可以带上 `-byres` 参数运行：

```bash
python auto_rename_ui.py -byres
```

### 支持的视频格式
程序支持处理主流视频格式：`.mp4`, `.avi`, `.mov`, `.mkv`, `.wmv`, `.flv`

## ⚙️ 处理工作流简述

1. **初始化缓冲**: 自动清理 `result` 文件夹下可能存在的残留旧文件。
2. **加载 AI 引擎**: 加载 EasyOCR 英文数字模型，屏蔽底层恼人的 C++ 警告输出。
3. **视频遍历与读取**: 读取视频并在 5秒、6秒、4秒处进行关键帧抽帧。
4. **OCR 识别与时间正则匹配**: 裁剪画面（250x1000区域），双倍放大后提取时间格式 (`YYYY...MM...DD...HH...MM...SS`)。
5. **多路处理与 FFmpeg 无损复制**: 计算准确时间后，调用后台 FFmpeg 将视频流无损 Copy (`-c copy`) 到新文件以保证极速，不进行重新编码。
6. **信息更新并展示**: 进度条持续推进，动态展示剩余数据量和估算时间。全部转换完成后弹窗提示。

## ⚠️ 注意事项 / Notes

- **覆盖提示**：每次运行程序前，`result` 目录下的原有文件会被清空，请注意提前备份。
- **UI 冻结防护**：繁重工作由后台 `threading` 托管，GUI 主线程专注刷新渲染，在处理大文件时不用担心应用窗口出现“无响应”。
- 防止 FFmpeg 在控制台刷屏打乱本地输出，程序将标准输出(stdout)以及标准错误(stderr)重定向到了 DEVNULL。
